MIT6.S081-Lab1 util: Unix utilities

实验目的

This lab will familiarize you with xv6 and its system calls.

本实验主要是编写几个用户态的小程序，小程序中会调用一些xv6中内置的系统调用，通过几个小程序的编写，来熟悉xv6中的系统调用。

sleep

Implement the UNIX program sleep for xv6; your sleep should pause for a user-specified number of ticks. A tick is a notion of time defined by the xv6 kernel, namely the time between two interrupts from the timer chip. Your solution should be in the file user/sleep.c.

思路

首先新建文件user/sleep.c，调用sleep系统调用实现睡眠功能，然后在Makefile的UPROGS中增加$U/_sleep\项(让make qemu将user/sleep.c编译为xv6中的程序sleep)。

pingpong

Write a program that uses UNIX system calls to ''ping-pong'' a byte between two processes over a pair of pipes, one for each direction. The parent should send a byte to the child; the child should print ": received ping", whereis its process ID, write the byte on the pipe to the parent, and exit; the parent should read the byte from the child, print ": received pong", and exit. Your solution should be in the file user/pingpong.c.

思路

首先新建文件user/pingpong.c。

调用pipe创建管道，调用fork创建子进程，父进程向管道write一个字节，然后wait(等子进程结束),子进程从管道read一个字节，print，再向管道write一个字节，父进程从管道read一个字节，print。

wait的存在使得父子进程各自读取对方发送的字节，不会出错。

最后在Makefile的UPROGS中增加$U/_pingpong\项(让make qemu将user/pingpong.c编译为xv6中的程序pingpong)。

primes

Write a concurrent version of prime sieve using pipes. This idea is due to Doug McIlroy, inventor of Unix pipes. The picture halfway down this page and the surrounding text explain how to do it. Your solution should be in the file user/primes.c.

思路

首先新建文件user/primes.c。

primes.c的主要作用是通过多进程实现一个高效的素数筛。原理如下：

本实验要求是筛出2..35中的素数，我们以2..35为例子。

定义一个进程它的父进程为左邻居，子进程为右邻居。（首进程无左邻居，尾进程无右邻居。）

首进程调用pipe创建管道，然后fork创建子进程(右邻居)，向自己创建的管道中写入2..35,然后wait等待子进程(右邻居)退出。

对于中间进程，以首进程的子进程(右边邻居)为例，它从父进程(左边邻居)创建的管道中读取数字，读取的第一个数字为2，然后pipe创建管道，fork创建子进程(右邻居)，向自己创建的管道中写入它从左边邻居(父进程)创建的管道中读取的不为2倍数的数字，然后wait 等待子进程(右边邻居)退出。

中间进程从左边邻居读取的第一个数字一定为素数(之间的进程没把它筛掉)，它要做的事情就是筛掉从左边邻居读取数字中为第一个数字倍数的数字，将不是第一个数字倍数的数字写到字节创建的管道中，传递给右边邻居。

尾进程只能从左边邻居(父进程)创建的管道中读取到一个素数，不需要再向右边邻居(子进程)传递任何数字。

最后在Makefile的UPROGS中增加$U/_primes\项(让make qemu将user/primes.c编译为xv6中的程序primes)。

find

Write a simple version of the UNIX find program: find all the files in a directory tree with a specific name. Your solution should be in the file user/find.c.

思路

首先新建文件user/find.c。

find.c的主要作用是从给定path中递归查找文件名为给定filename的文件，并打印该文件的路径。

思路很简单：打开目录文件path,遍历path中的目录项，由目录项的文件名拼接path获取文件的完整路径名，由文件完整路径名获取文件信息，判断文件是否为目录文件，若是，递归遍历；若不是，判断文件名字是否==filename,若是，打印文件完整路径名。可参考一下user/ls.c文件。

最后在Makefile的UPROGS中增加$U/_find\项(让make qemu将user/find.c编译为xv6中的程序find)。

xargs

Write a simple version of the UNIX xargs program: read lines from the standard input and run a command for each line, supplying the line as arguments to the command. Your solution should be in the file user/xargs.c.

思路

首先新建文件user/xargs.c。

xargs.c的输入是一条命令，xargs.c的作用是从输入中每次读取一行，读取的一行作为输入命令的参数，然后执行这条命令。读取的每一行都作为输入命令的参数执行一次。

最后在Makefile的UPROGS中增加$U/_xargs\项(让make qemu将user/xargs.c编译为xv6中的程序xargs)。
