MIT6.S081-Lab10 mmap: Mmap

实验目的

The mmap and munmap system calls allow UNIX programs to exert detailed control over their address spaces. They can be used to share memory among processes, to map files into process address spaces, and as part of user-level page fault schemes such as the garbage-collection algorithms discussed in lecture. In this lab you'll add mmap and munmap to xv6, focusing on memory-mapped files.

本实验主要实现两个系统调用：

  1. sys_mmap: 将一个文件映射到用户虚拟地址空间的某个地址处。
  2. sys_munmap：取消用户虚拟地址空间中某个地址范围内的映射。

mmap

mmap()函数的声明如下：

  • addr: 表示需要映射到用户虚拟地址空间中的位置。本实验中addr==0，由系统自己决定映射的位置(addr)。

    mmap()的返回值为映射的位置，0xffffffffffffffff(即-1)代表mmap()失败。

  • length: 表示需要映射的字节数。即把文件映射到地址范围[addr,addr+length)中，length可能与文件长度不同。

  • prot: 表示映射的地址范围的权限(read、write、executable)，由prot来决定页表项中的标志位。

  • flags: 表示取消映射时是否把内存中映射范围内的数据(已修改)写回到磁盘文件中。

    • MAPSHARED: 写回
    • MAP_PRIVATE: 不写回

  • fd: 表示想要映射文件的文件描述符。

  • offset: 表示文件的偏移量，从文件开始偏移offset个字节处开始映射文件内容。本实验中offset == 0，一直从文件头开始映射。

munmap

munmap()函数的声明如下：

  • addr: 表示取消映射的起始地址。
  • length: 表示取消映射的地址长度(byte)。

即取消映射的地址范围为[addr, addr + length）。

注意：

假定一个文件现在还映射在虚拟地址空间中的虚拟地址范围为[start, end), munmap()参数必然有以下两种限制之一

  • addr == start && addr + length <= end
  • addr >= start && addr + length == end

即取消映射的范围必须跟[start, end)左对齐或右对齐，不允许在[start, end）中间取消映射一部分，保证文件仍映射在虚拟地址空间中的范围始终是连续的。
