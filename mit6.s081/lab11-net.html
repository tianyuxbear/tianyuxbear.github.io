MIT6.S081-Lab11 net: Network stack

实验目的

In this lab you will write an xv6 device driver for a network interface card (NIC).

QEMU模拟了一个用于网络通信的硬件E1000，QEMU同时模拟了一个LAN,在模拟的局域网中，xv6所在的虚拟主机的IP为 10.0.2.15("guest")，我们执行make qemu的物理主机的IP为10.0.2.2("host")。QEMU模拟"guest"和“host”之间的网络通信。

We've added some files to the xv6 repository for this lab. The file kernel/e1000.c contains initialization code for the E1000 as well as empty functions for transmitting and receiving packets, which you'll fill in. kernel/e1000_dev.h contains definitions for registers and flag bits defined by the E1000 and described in the Intel E1000 Software Developer's Manual. kernel/net.c and kernel/net.h contain a simple network stack that implements the IP, UDP, and ARP protocols. These files also contain code for a flexible data structure to hold packets, called an mbuf. Finally, kernel/pci.c contains code that searches for an E1000 card on the PCI bus when xv6 boots.

Your job is to complete e1000_transmit() and e1000_recv(), both in kernel/e1000.c, so that the driver can transmit and receive packets. You are done when make grade says your solution passes all the tests.

函数e1000_init()初始化硬件E1000,硬件E1000主要完成两项工作：

  1. read: 读取要从RAM传输的数据包("guest" transmit)。

e1000_init()为硬件E1000初始化了一个transmit queue，即struct tx_desc(描述transmit packet的信息)的数组tx_ring，同时初始化了一个packet buffers,即struct mbuf*(RAM中保存transmit packet的位置)的数组tx_mbufs。

  1. write: 写入RAM接收到的数据包("guest" receive)。

e1000_init()为硬件E1000初始化了一个receive queue，即struct rx_desc(描述receive packet的信息)的数组rx_ring，同时初始化了一个packet buffers,即struct mbuf*(RAM中保存receive packet的位置)的数组rx_mbufs。

e1000_transmit()

When the network stack in net.c needs to send a packet, it calls e1000_transmit() with an mbuf that holds the packet to be sent.

全局变量regs指向E1000的第一个控制寄存器，可以把regs看作一个数组，用宏定义去索引对应控制寄存器的值。int index = regs[E1000_TDT];

e1000_transmit()的作用：找到tx_ring[index]对应tx_desc，判断这个tx_desc上一次transmit 工作是否完成(看状态中是否有E1000_TXD_STAT_DD标志位), 如果上一次transmit已完成，则根据准备transmit的mbuf修改tx_desc的信息，同时把tx_mbufs[index]更新为准备transmit的mbuf,丢弃上一次transmit的mbuf。

e1000_recv()

When the E1000 receives each packet from the ethernet, it first DMAs the packet to the mbuf pointed to by the next RX (receive) ring descriptor, and then generates an interrupt.

全局变量regs指向E1000的第一个控制寄存器，可以把regs看作一个数组，用宏定义去索引对应控制寄存器的值。

int index = (regs[E1000_RDT] + 1) % RX_RING_SIZE; // + 1是对应题干中的 the next

e1000_recv()的作用：找到rx_ring[index]对应rx_desc, 判断这个rx_desc上一次recv工作是否完成(看状态中是否有E1000_RXD_STAT_DD标志位)，如果上一次recv工作已完成，则把rx_mbufs[index]对应的接收到的mbuf信息通过net_rx()传输到网络栈中。然后申请一个新的mbuf，放到rx_mbuf[index]以便下次循环接收packet到index对应位置时，有新的mbuf可以receive packet。
