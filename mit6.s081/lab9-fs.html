MIT6.S081-Lab9 fs: File system

实验目的

In this lab you will add large files and symbolic links to the xv6 file system.

Large files

In this assignment you'll increase the maximum size of an xv6 file. Currently xv6 files are limited to 268 blocks, or 268*BSIZE bytes (BSIZE is 1024 in xv6). This limit comes from the fact that an xv6 inode contains 12 "direct" block numbers and one "singly-indirect" block number, which refers to a block that holds up to 256 more block numbers, for a total of 12+256=268 blocks.

本实验主要是修改dinode的结构，使得dinode对应的文件能包含更多的disk block,增加文件的容量。xv6中每个磁盘上的文件都有一个dinode结构体来保存它的元信息。

数组addrs中每个uint值都是一个block number，其中前NDIRECT个uint值用于索引dinode的一个block data，最后一个uint值同样索引一个block data，只是这个block data中的数据不是dinode对应的文件数据，而是256个uint值(block size = 1024), 这256个uint值用于索引dinode的一个block data。

所以，dinode中的addrs一共可以索引 NDIRECT + BSIZE(block size) / sizeof(uint)个block,即 12 + 1024 / 4 = 12 + 256 = 268个block，dinode对应的文件大小最大为268blocks。

Modify bmap() so that it implements a doubly-indirect block, in addition to direct blocks and a singly-indirect block. You'll have to have only 11 direct blocks, rather than 12, to make room for your new doubly-indirect block; you're not allowed to change the size of an on-disk inode. The first 11 elements of ip->addrs[] should be direct blocks; the 12th should be a singly-indirect block (just like the current one); the 13th should be your new doubly-indirect block. You are done with this exercise when bigfile writes 65803 blocks and usertests runs successfully:

明确实验目的后，按照hints一步步做就可以了。

  • 修改struct dinode，增加文件的容量。(kernel/fs.h)

    注意struct dinode结的大小要保持不变，为64(4 * 2 + 4 + 13 * 4)个byte。因为struct dinode保存在disk block中，每个block包含16个struct dinode，查找文件时首先从它的目录项中获取它的dinode编号，根据dinode编号去disk中索引一个struct dinode，再根据dinode中的addrs索引对应的block data(文件数据)。struct dinode的大小改变会破坏这个设定。

    修改后 NDIRECT = 11，NINDIRECT = (BSIZE / sizeof(uint)), NDINDIRECT = (NINDIRECT * NINDIRECT) 。

    • addrs中前11个uint: 直接索引，索引 NDIRECT(11) 个block data。
    • addrs中第12个uint: 单步间接索引，索引 NINDIRECT(256) 个 block data.
    • addrs中第13个uint: 双重间接索引，索引 NDINDIRECT个(256 * 256) block data.

    现在一个dinode中可以索引 11 + 256 + 256 * 256 个block data, 即文件的最大容量为65803blocks.

Symbolic links

You will implement the symlink(char *target, char *path) system call, which creates a new symbolic link at path that refers to file named by target. For further information, see the man page symlink. To test, add symlinktest to the Makefile and run it. Your solution is complete when the tests produce the following output (including usertests succeeding).

本实验的目的是实现一个创建符号链接的系统调用，用path作为完整路径文件名创建一个文件，指明它的文件类型为符号链接，这个文件的inode中只有一个数据块号,用于保存链接文件的完整路径文件名target。当read或write一个符号链接文件时，根据它数据库中保存的链接文件的完整路径名字，查找到对应的非符号链接文件，再进行read或write操作。

符号链接又称为软链接。

相对应的有硬链接。比如/x/a1和/y/a2是一对硬链接文件，目录/x/中有文件/x/a1的目录项，此目录项中的文件名为a1，inode号为1234(随便编的),目录/y/中有文件/y/a2的目录项，此目录项中的文件名为a2，inode号也为1234。即/x/a1和/y/a2看上去是两个文件，实际上对应同一个inode编号，磁盘上只有同一份内容。
